Services

Service: AnalyzerController

O AnalyzerController deve ser implementado pela app e seus métodos são chamados pelo Radar. Suas funções principais são registrar os analyzers, controlar o ciclo de vida das sessões dos analyzers e repassar os eventos para os mesmos. Método que registra os analyzers: analyzers Métodos que controlam o ciclo de vida da sessão: create_session (criação), result (destruição) Métodos que devem repassar o controle para os analyzers: on_each_day, on_each_month, on_finish, on_cash_flow, dump, resume, result, example_result

Function: AnalyzerController.dump

binary dump(SessionId session_id)
É chamado sempre que o Radar quiser gerar uma imagem do estado atual do serviço para poder restaurar nos próximos processamentos. O retorno pode ter qualquer formato, desde que seja compreendido pelo método resume(), já que posteriormente o conteúdo do retorno será passado como argumento para o método.
